# 프론트엔드 수정 가이드 (웹캠 + 아두이노 연동)

> Python AI 서버(EC2)와 연동하기 위해 프론트엔드에서 수정해야 할 내용입니다.
> 현재 프론트엔드에는 웹캠 관련 코드가 전혀 없습니다. 새로 추가해야 합니다.

---

## 전체 흐름

```
[브라우저]
  1. 학습 시작 버튼 클릭
       → Spring: POST /api/v1/sessions (세션 생성)
       → Python EC2: POST /vision/start (분석 준비)
       → 웹캠 켜기
       → 1초마다 프레임 캡처 → Python EC2로 전송
       → 분석 결과(졸음/이탈/집중) 수신
       → 졸음/이탈이면 Web Serial API로 아두이노에 신호 전송

  2. 학습 종료 버튼 클릭
       → 웹캠 끄기
       → Python EC2: POST /vision/stop
       → Spring: POST /api/v1/sessions/{id}/end
```

## 2. Python EC2 API 서비스 (`src/services/vision.js`) 신규 생성

Python AI 서버와 통신하는 axios 호출 모음.

```js
const PYTHON_AI_URL = import.meta.env.VITE_PYTHON_AI_URL;

// 분석 시작
// POST /vision/start  { sessionId, userId }
export const startVision = (sessionId, userId) => { ... }

// 프레임 전송 + 분석 결과 수신
// POST /vision/frame  multipart/form-data  field: "file" (JPEG Blob)
// 응답: { focusState: "GOOD" | "DROWSY" | "ABSENT", elapsed: float, isTimerActive: bool }
export const sendFrame = (frameBlob) => {
  const formData = new FormData();
  formData.append("file", frameBlob, "frame.jpg");
  return axios.post(`${PYTHON_AI_URL}/vision/frame`, formData);
}

// 분석 종료
// POST /vision/stop  { sessionId }
export const stopVision = (sessionId) => { ... }
```

> ⚠️ Python EC2가 현재 HTTP라서 브라우저에서 직접 호출 시 Mixed Content 오류 발생 가능.
> Vercel(HTTPS) → Python EC2(HTTP) 호출은 브라우저가 차단합니다.
> 해결 방법은 하단 "인프라 이슈" 참고.

**`sendFrame` 응답 필드 설명:**

| 필드 | 타입 | 설명 |
|---|---|---|
| `focusState` | `"GOOD"` \| `"DROWSY"` \| `"ABSENT"` | 현재 확정된 집중 상태 |
| `elapsed` | `number` | 위반 상태 지속 시간(초). 정상이면 0 |
| `isTimerActive` | `boolean` | 위반 타이머 진행 중 여부 |

## 3. 웹캠 훅 (`src/hooks/useWebcam.js`) 신규 생성

브라우저 웹캠을 켜고, 일정 간격으로 프레임을 캡처하는 훅.

```
기능:
  - startWebcam(): navigator.mediaDevices.getUserMedia()로 웹캠 시작
  - stopWebcam(): 스트림 종료
  - captureFrame(): 현재 프레임을 canvas에 그려서 base64 JPEG로 반환
  - videoRef: <video> 태그에 연결할 ref
```

## 5. 집중도 모니터링 훅 (`src/hooks/useFocusMonitor.js`) 신규 생성

웹캠 + Python EC2 분석 + 아두이노 알림을 총괄하는 메인 훅.

```
흐름:
  start(sessionId, userId):
    1. useWebcam.startWebcam()
    2. vision.startVision(sessionId, userId)
    3. setInterval로 1초마다:
         frameBlob = useWebcam.captureFrame()   // canvas → Blob(JPEG)
         result = vision.sendFrame(frameBlob)
         setCurrentState(result.focusState)     // UI 상태 업데이트
         // ESP32 알림은 Python EC2가 자동으로 처리 — 프론트 불필요

  stop(sessionId):
    1. clearInterval
    2. vision.stopVision(sessionId)
    3. useWebcam.stopWebcam()

반환값:
  { start, stop, currentState, isMonitoring }
  currentState: "GOOD" | "DROWSY" | "ABSENT"
```

## 7. Editor.jsx 수정

현재 `Editor.jsx`의 학습 시작/종료 로직:

```js
// 현재 코드 (session.js만 호출)
const handleLearningToggle = async () => {
  if (!isLearning) {
    const { sessionId: sid } = await startSession();
    setSessionId(sid);
    setIsLearning(true);
  } else {
    await endSession(sessionId);
    setSessionId(null);
    setIsLearning(false);
  }
};

  } else {
    await stopMonitor(sessionId);      // ← 추가
    await endSession(sessionId);
    setSessionId(null);
    setIsLearning(false);
  }
};

// JSX에 FocusMonitor 컴포넌트 추가
{isLearning && <FocusMonitor state={currentState} />}

## 인프라 이슈 (반드시 해결 필요)

### 문제: Mixed Content 오류

Vercel은 HTTPS, Python EC2는 HTTP → 브라우저가 차단

### 해결 방법 선택지

**옵션 1 — Python EC2에 HTTPS 적용 (권장)**

```
도메인 연결 + Let's Encrypt SSL 인증서 발급
예: python-ai.aice-edu.site → Python EC2 IP
```

**옵션 2 — Spring이 Python API를 프록시**

```
프론트 → Spring (HTTPS) → Python EC2 (HTTP)
Spring에 /api/v1/vision/* 엔드포인트 추가해서 Python으로 중계
```

**옵션 3 — Python EC2 포트를 Nginx로 HTTPS 처리**

```
EC2에 Nginx + SSL 설정
443(HTTPS) → 5000(Python FastAPI) 프록시
```
